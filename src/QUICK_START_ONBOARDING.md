# Quick Start: Private Access & Onboarding System

## What Was Built

A **complete private-access, approval-based onboarding system** for the Reentry & Jobs platform.

**No public access.** Everyone must either:
1. **Log in** (existing approved users), or
2. **Request access** (new users submit intake/request form)

Admin approves all requests before users get system access.

---

## 🚀 For Users: Getting Started

### New Resident

1. Visit the app URL → See **Public Landing Page**
2. Click **"Request Access"** → Select **"Resident / Participant"**
3. Fill out **Resident Intake Form** (name, email, housing, employment, needs, etc.)
4. Submit → Request appears in **Admin Queue**
5. Wait for **admin approval** (AI analysis runs automatically)
6. Receive **approval email** with temporary login code
7. Click **"Log In"** on public page
8. Enter email + temporary code from email
9. Create permanent password
10. Access **Resident Dashboard** (see only your own data)

### New Staff / Case Manager / Employer / Partner

1. Visit app URL → Click **"Request Access"**
2. Select your role (Case Manager, Employer, etc.)
3. Fill out **Role Request Form** (name, email, organization, reason)
4. Submit → Request appears in **Admin Queue**
5. Wait for admin approval
6. Receive approval email with temporary code
7. Log in with temporary code
8. Set permanent password
9. Access your **role-specific dashboard**

### Existing Approved User

1. Visit app → Click **"Log In"**
2. Enter email + password
3. Access your dashboard immediately
4. Stay private—see only data you're authorized for

## 🛠️ For Admins: Managing Access

### Review Pending Requests

1. Log in with **admin role**
2. Click **"Onboarding Queue"** in sidebar (Admin section)
3. See **Pending** tab with new requests
4. Click any request to view full details

### What You'll See

- **Applicant info** (name, email, phone, organization)
- **Requested role** (what they asked for)
- **AI Summary** (AI analysis of their request—*advisory only*)
- **AI Recommended Role** (AI suggestion—*you override if needed*)
- **AI Recommended Services** (for residents: suggested classes)

### Approve a Request

1. Open request detail view
2. Select **final role** dropdown (you can override AI recommendation)
3. Optionally add **admin notes**
4. Click **"Approve Request"**
5. ✅ System creates:
   - User account
   - Resident/Employer record (if applicable)
   - Temporary login code
6. 📧 Sends approval email with code to applicant

### Reject a Request

1. Open request detail view
2. Click **"Reject Request"**
3. Enter reason for rejection
4. Click confirm
5. ❌ Request marked as rejected, no account created
6. 📧 (Optional: send rejection email to applicant)

### Request More Information

1. Open request detail view
2. Move to **"Needs Info"** tab (or set status manually)
3. Reach out to applicant directly to collect more details
4. When info received, reopen request and approve

### Manage Existing Users

1. Go to **"User Management"** → Users
2. View all users with their status:
   - **pending** → admin just approved
   - **invited** → activation email sent
   - **active** → logged in, full access
   - **suspended** → temporarily blocked
   - **deactivated** → permanently blocked
3. Actions per user:
   - ✏️ Edit details / change role
   - 🔄 Resend activation email
   - 🔐 Reset password (regenerate temporary code)
   - 🚫 Suspend or deactivate user
   - ✅ Activate user

## 📋 Key Concepts

### Temporary Login Code

- 12-character code sent in approval email
- Valid for **7 days**
- User enters on first login to verify email ownership
- After code entry, user sets permanent password
- Can be reset/regenerated by admin anytime

### Statuses

**OnboardingRequest Status:**
- `pending` → awaiting admin review
- `needs_info` → admin requested more details
- `approved` → admin approved, user account created
- `rejected` → admin rejected, no account created

**UserAccount Status:**
- `pending` → created, awaiting activation
- `invited` → activation email sent
- `active` → full access granted
- `suspended` → temporarily blocked
- `deactivated` → permanently disabled

### AI Analysis

- **Runs automatically** after request submission
- **Summarizes** the request narrative
- **Recommends role** (for help, not binding)
- **Suggests services/classes** (for residents)
- **Flags concerns** (data quality issues, red flags)
- **Admin makes final decision**—AI is advisory only

## 🎯 Common Tasks

### I'm a new resident. How do I get access?

1. Go to app → "Request Access"
2. Fill out intake form
3. Wait for admin to approve (1-2 business days)
4. Check email for approval + login code
5. Log in with code, set password
6. You're in!

### I'm an employer. How do I post jobs?

1. Go to app → "Request Access"
2. Select "Employer" role
3. Fill out organization and reason
4. Wait for admin approval
5. Log in with temporary code
6. Access Employer Portal → "My Jobs & Candidates"
7. Post jobs and review applications

### I forgot my password. What do I do?

1. On login page, use password reset option (built-in)
2. Or: Admin can reset your password from User Management
3. You'll get an email with temporary code
4. Use code to set new permanent password

### How do I deactivate a user?

1. Go to User Management (`/users`)
2. Find user in list
3. Edit user → change Status to "deactivated"
4. Save
5. User can no longer log in
6. (To reactivate later: change Status to "active")

### I'm an admin. How do I invite someone without the intake form?

1. Go to User Management (`/users`)
2. Click "Invite User" (or create directly)
3. Enter email, select role
4. User gets activation email
5. They log in with temporary code
6. They set permanent password
7. Done! (No OnboardingRequest needed)

## 🚨 Troubleshooting

### User didn't receive approval email

**Action:**
1. Go to User Management
2. Find user
3. Click "Resend Activation Email"
4. Check spam folder (email should arrive within seconds)

### User's temporary code expired

**Action:**
1. Go to User Management
2. Find user
3. Click "Reset Password"
4. New temporary code generated
5. Resend activation email automatically

### Resident can see other residents' data

⚠️ **Security Issue!**  
This should not happen. Verify:
1. Backend functions use `queryResidentsScoped()`
2. Database row-level security (RLS) is configured
3. Resident lookup by email is working
4. Contact support if issue persists

### User locked out / can't log in

**Check:**
1. Is user status "active"? (If suspended/deactivated, change it)
2. Is their role correct?
3. Did they set a permanent password? (If not, reset via admin)
4. Are they using correct email?

## 🎓 System Behavior Examples

### Example 1: New Resident Intake

1. Sarah visits app → sees "Request Access" button
2. Clicks it → selects "Resident"
3. Fills intake: name, email, housing, employment, needs
4. Submits
5. AI analysis runs (summary, recommended role, suggested services)
6. Admin John reviews in queue → sees AI analysis
7. John approves → creates resident + user account
8. Sarah gets email: "Approved! Temporary code: ABC123DEF456"

10. Forced to set password
11. Logged into Resident Dashboard (sees only her data)

### Example 2: New Employer

1. Mike's company wants to post jobs
2. Mike visits app → "Request Access" → "Employer"
3. Fills: company name, email, reason

5. Enters reason: "Insufficient information provided"
6. Request marked as rejected
7. Applicant *does not* get account
8. (Optional: send rejection email explaining why)
